# Agent instructions

Before changing code:
1. Read `docs/PRODUCT.md`, `docs/PRE_MVP_SCOPE.md`, `docs/ARCHITECTURE.md` and the relevant feature specification.
2. State assumptions.
3. Propose the smallest verifiable implementation.
4. List files to be changed.
5. Wait for approval only when the requested action is destructive or requires credentials.

After changing code:
1. Run typecheck, lint and build.
2. Summarize changed files.
3. Explain manual test steps.
4. Report unresolved risks honestly.
5. Suggest one next task only.

Never generate, guess or commit credentials.
