# Together Pre-MVP Starter

Готовая стартовая основа мобильного PWA для пар.

## Что внутри

- минимальный Next.js-интерфейс;
- TypeScript и Tailwind CSS;
- заготовки Supabase browser/server clients;
- начальная SQL-схема;
- базовые RLS-политики;
- seed-вопросы;
- Cursor Rules и `AGENTS.md`;
- документация продукта;
- инструкции локального запуска, GitHub, Supabase и Vercel;
- готовые промпты для Cursor.

## Быстрый старт

```bash
cp .env.example .env.local
npm install
npm run dev
```

Открой `http://localhost:3000`.

Пока `.env.local` пустой, демонстрационные страницы работают, но реальная авторизация и база не подключены.

## Правильный порядок

1. Прочитать `docs/START_HERE.md`.
2. Открыть проект в Cursor.
3. Вставить в Cursor содержимое `prompts/00_CURSOR_PROJECT_AUDIT.md`.
4. Запустить проект локально.
5. Создать Supabase и применить миграцию.
6. Создать приватный GitHub-репозиторий.
7. Развернуть проект на Vercel.
8. Реализовывать функции по одной.

## Важное предупреждение

SQL в архиве — безопасная стартовая база, а не доказательство полной безопасности. До приглашения реальных пользователей нужно вручную протестировать изоляцию данных минимум с тремя аккаунтами: двумя участниками пары и одним посторонним.
