# Prompt: deploy to Vercel

Inspect the Next.js project and `.env.example`.

Guide me through:
1. Importing the private GitHub repository into Vercel.
2. Confirming the detected framework and root directory.
3. Adding all required environment variables to Development, Preview and Production.
4. Deploying.
5. Adding the production URL to Supabase Auth redirect URLs.
6. Triggering a clean redeploy.
7. Testing the production URL on a phone.
8. Explaining preview deployments for feature branches.

Never ask for or expose credentials. Do not use a service role key in Vercel unless a later server-only feature explicitly requires it.
