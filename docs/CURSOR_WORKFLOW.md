# Cursor workflow

## Один цикл

1. Создать feature branch.
2. Дать Cursor прочитать документацию.
3. Попросить план без кода.
4. Уточнить одну маленькую функцию.
5. Реализовать.
6. Выполнить `npm run check`.
7. Проверить вручную.
8. Посмотреть diff.
9. Commit.
10. Push и Vercel Preview.

## Хорошая задача

«Добавь пустое состояние списка планов. Не подключай базу. Не меняй навигацию».

## Плохая задача

«Сделай всё приложение красивым, добавь чат, базу, авторизацию и игры».

## Обязательная проверка diff

```bash
git status
git diff
git diff --staged
```

Cursor может уверенно предложить неправильное решение. Проверка результата остаётся обязанностью разработчика.
