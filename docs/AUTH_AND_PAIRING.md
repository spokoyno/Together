# Authentication and pairing

## Auth

Для первого теста:
- email + password или magic link;
- подтверждение email;
- безопасная SSR-сессия через cookie;
- redirect URLs для localhost, Preview и Production.

## Pairing

1. Первый пользователь создаёт пару.
2. Сервер создаёт криптографически случайный токен.
3. В БД сохраняется хэш токена.
4. Пользователь отправляет ссылку партнёру.
5. Сервер проверяет хэш, срок и состояние приглашения.
6. Сервер проверяет, что принимающий пользователь не состоит в другой паре.
7. Создаётся membership.
8. Приглашение помечается принятым.

## Запрещённый упрощённый вариант

Нельзя разрешать браузеру свободно вставлять `couple_members` для произвольного `couple_id`. Этот сценарий должен быть реализован доверенной серверной операцией.
