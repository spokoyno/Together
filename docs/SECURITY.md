# Security and privacy baseline

## Обязательные правила

- RLS включён на каждой таблице `public`.
- Посторонний пользователь не видит данные пары.
- Service role key никогда не отправляется в браузер.
- В Git нет `.env.local`.
- Приложение не логирует личные ответы и настроение.
- Медиа хранятся в приватном bucket.
- Signed URLs имеют ограниченный срок.
- Удаление и отвязка требуют повторного подтверждения.
- Экспорт доступен только владельцу данных или члену пары в разрешённом объёме.

## Тест изоляции

Создай пользователей A, B и C.

1. A и B состоят в паре.
2. A создаёт план и воспоминание.
3. B может их прочитать.
4. C не может прочитать, изменить или удалить их.
5. A не может подставить `created_by = B`.
6. После выхода токен больше не даёт доступ.
7. После отвязки доступ пересчитывается согласно принятой политике хранения.

## Особо чувствительные будущие функции

- геолокация;
- тревожная кнопка;
- цикл;
- психологические ответы;
- финансы;
- документы;
- дети.

Они не входят в Pre-MVP без отдельной модели угроз и согласия пользователя.
