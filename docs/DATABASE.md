# Database model

## profiles

Публично-минимальная пользовательская карточка, связанная с `auth.users`.

## couples

Общее пространство отношений.

## couple_members

Связь пользователя с парой. Ограничение `unique(user_id)` сохраняет правило одной активной пары.

## invitations

Приглашения партнёра. В финальной реализации исходный токен не должен храниться открытым текстом.

## moods

История добровольно опубликованного настроения.

## plans

Универсальный список общих планов.

## memories

Текстовые и медиа-воспоминания. `media_path` должен хранить путь объекта, а не публичный бессрочный URL.

## questions, daily_questions, answers

Каталог вопросов, выбранный вопрос пары на дату и приватные ответы.

## events

Годовщины, встречи и другие даты.

## Миграции

Начальная схема находится в:

```text
supabase/migrations/001_initial_schema.sql
```

Применяй её через Supabase SQL Editor только после чтения и понимания.

## Ограничение стартовой схемы

Процесс принятия приглашения требует отдельной безопасной серверной функции или RPC. Не реализуй его прямой вставкой из браузера без проверки токена, срока действия и текущего членства.
